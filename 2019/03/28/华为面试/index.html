2019-3-27华为实习春招笔试

发表于 2019-03-28 | 更新于 2019-03-29 | 分类于 algorithmn | 阅读次数：
本文字数： 3.2k | 阅读时长 ≈ 3 分钟

华为3道笔试题，2个小时，共600分，做出了2道，有点虚。第一题是送分题，最后一道没有思路，有想法后再更新。

第一题

关于大小端的问题，一串字符串长9个字符，第一个字符为‘0’则代表是小端，第一个字符为‘1’则代表是大端，给你一个数字表示字符串的个数，和一行字符串（全部字符串连接在一起了），输出对应数据的大端表示，每串字符串之间有一个空格。

例子

解决方法

这题是比较简单，其实大端小端我都记得不太清楚，虽然经常接触。但在这里只要简单地实现为当第一个字符为’0’时，将字符串倒序输出，当第一个字符为’1’时，将字符串输出。

第二题

关于旅行商问题的，给你5个坐标，表示5个地方的位置，从（0，0）出发，计算访问5个顶点然后回到原点的最短距离。输入的点是不会重复的。

时间限制：3s

例子

这题是想得比较久的，这在之前的算法课是相似的，只不过不是5个坐标，而是N个坐标。如果是N个坐标，这就是著名的旅行商的问题。这个是NP难问题。我的天，不会想让我用模拟退火法或者遗传算法之类的人工智能算法吧！

只给我3s，我不知道能不能用模拟退火法跑出结果。之前在比较大的数据集里写过这个算法，我不确定能不能得到准确的结果。因此在笔试中，试图用其他方法来解。

解决方法

最终还是选择用暴力破解。总共才5 4 3 2 1 = 120种方法，只要全部遍历，记录最短的，返回就好了。

注意的是，还要加上原点，而且用一个2 * 2的数组来记录距离。

使用死亡5循环，无论怎样，还是过了。 算法复杂度怎么说也是 $O(1)$.

第三题

切水果问题

一张图大小40 * 50, 给一个N值(0 < N < 36), 然后给出N个点（水果），输出最少能用多少条线使得水果消失。

一个点只有4种切法，横、竖以及两条对角线。

左上角是(0, 0)，右下角是(39, 49)。(感觉不重要)

例子

想了一个多小时，想不出怎么做，牛客网里有人用贪心算法做，过了80%。就是每次选最多点共线的线，但是解决不了4个点组成’Z’型的情况。如果先选竖，则要切3刀，但先选横，只要2刀。
